📘 How to write blocks & use variables

A block is a fragment of email-safe HTML — usually one or more <tr>...</tr> rows or a self-contained <table>. Blocks compose into templates via the visual builder.

Variables

Always use the dot-prefixed Go template syntax:

  • {{"{{.Variable}}"}} — substitutes a value from the payload.
  • {{"{{if .Cond}}...{{end}}"}} — renders the inside only when the value is truthy.
  • {{"{{range .Items}}{{.Name}}: {{.Price}}{{end}}"}} — iterates over an array of objects.

Example payload for a block using {{"{{range .Items}}"}}:

{
  "OrderNumber": "12345",
  "Items": [
    {"Name": "Product A", "Price": "100 ₽"},
    {"Name": "Product B", "Price": "200 ₽"}
  ]
}

HTML rules

  • Use tables for layout — Outlook ignores flex/grid.
  • Inline styles only — <style> blocks are stripped by Gmail/Yandex.
  • Set explicit width on tables; cap visible content at 600px.
  • Use role="presentation" on layout tables for accessibility.

Per-block appearance

Border-radius, padding, and background colour are set on the block's view page (Appearance card). They wrap the block at render time so the source HTML stays clean.

Inline editing

Plain text inside blocks can be edited directly on the canvas in the builder. Variables and surrounding markup are protected — edit the placeholder names through the block's HTML source instead.
